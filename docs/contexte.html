Contexte et Formulation¶

Contexte applicatif¶

Dans de nombreux scénarios de cyberdéfense, il est nécessaire de déployer un nombre limité de ressources de surveillance afin de couvrir un ensemble de cibles critiques :

  • Sondes IDS — détection d’intrusions sur segments réseau

  • Règles de filtrage — pare-feux et ACL sur flux entrants/sortants

  • Outils d’analyse de vulnérabilités — scanners, agents d’audit

Le déploiement est contraint par un budget : il est impossible d’activer toutes les ressources disponibles simultanément.

Ce problème appartient à la famille des problèmes de couverture maximale (Maximum Coverage Problem), connu pour être NP-difficile, ce qui justifie le recours aux heuristiques et métaheuristiques.

Formulation mathématique¶

Données du problème

Étant donné :

  • \(U = \{u_1, \dots, u_m\}\) — un ensemble de m cibles à couvrir

  • \(S = \{S_1, \dots, S_n\}\) — un ensemble de n ressources, chacune couvrant un sous-ensemble de cibles \(S_i \subseteq U\)

  • \(c_i > 0\) — le coût d’activation de la ressource \(S_i\)

  • \(B > 0\) — le budget total disponible

Variables de décision

\[ \begin{align}\begin{aligned}x_i \in \{0, 1\},\quad i = 1, \dots, n \qquad \text{(ressource i sélectionnée ?)}\\y_j \in \{0, 1\},\quad j = 1, \dots, m \qquad \text{(cible j couverte ?)}\end{aligned}\end{align} \]

Objectif lexicographique

\[\max \left( \sum_{j=1}^{m} y_j,\ -\sum_{i=1}^{n} c_i x_i \right)\]

  1. Critère principal : maximiser le nombre de cibles couvertes \(\sum_{j} y_j\)

  2. Critère secondaire : à égalité, minimiser le coût total \(\sum_{i} c_i x_i\)

Contraintes

\[ \begin{align}\begin{aligned}\sum_{i=1}^{n} c_i x_i \leq B \qquad \text{(contrainte de budget)}\\y_j \leq \sum_{i \,:\, u_j \in S_i} x_i \qquad \forall j \in \{1,\dots,m\}\end{aligned}\end{align} \]

Complexité

Le problème est NP-difficile par réduction depuis le Set Cover Problem. Pour \(m\) cibles et \(n\) ressources, l’espace de solutions feasibles est de taille \(O(2^n)\), rendant toute énumération exhaustive impossible au-delà de quelques dizaines de ressources.

Format des instances¶

Les fichiers d’instance suivent le format texte suivant :

n m B
c_1 c_2 ... c_n
k_1 u_11 u_12 ...
k_2 u_21 u_22 ...
...
Description des champs¶

Symbole

Signification

n

Nombre de ressources disponibles

m

Nombre de cibles à couvrir

B

Budget total

c_i

Coût d’activation de la ressource i

k_i

Nombre de cibles couvertes par la ressource i

u_ij

Indice (base 0) de la j-ième cible couverte par i
